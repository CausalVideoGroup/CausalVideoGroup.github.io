07A · Long horizon / rollout side

先控制误差怎么传播，再让历史保持动态

这组方法不换教师，而是直接修改生成过程与长期锚点。

Generation process

Rolling Forcing

问题：严格逐帧/逐块因果会把上一时刻误差立刻传给下一时刻。
不要一次只完成一帧：让滚动窗口中的多帧一起去噪，局部放松严格因果。
  • 窗口内多帧联合去噪
  • 未来方向使用渐进升高的噪声
  • 首帧 attention sink 保留全局锚点
  • 在扩展窗口上做少步蒸馏
结果：抑制误差增长，支持实时分钟级生成
Dynamic anchor + objective

Reward Forcing

问题：固定首帧 sink 虽然稳定，却让后续帧复制开头、运动逐渐消失。
锚点也要随视频更新：长期记忆不能永远停在第一帧。
  • EMA-Sink 融合滑窗中被移出的历史 token
  • 固定缓存规模，同时吸收近期动态
  • Re-DMD 用视觉语言模型的动态奖励重加权样本
结果：稳定性不丢，同时减少首帧复制与运动衰减

短教师如何教一个比自己长几十倍的学生？

Long rollout → short correction

Self-Forcing++

问题：短视频 Teacher 无法直接生成或监督几分钟的完整轨迹。
学生负责走远，教师只负责局部纠偏：从学生的长 rollout 中抽短窗口。
  • 学生先生成超出训练长度的长视频
  • 随机抽取短片段送入短时双向 Teacher
  • Teacher 修正局部窗口，再蒸馏回学生
  • Backward-noise initialization：对抽取的学生 latent 重新加噪，保留它与前文的内容关联
不需要真实长视频数据或长视频 Teacher
Long teacher ↔ long student

Context Forcing

问题：Teacher 只看短片段，而 Student 根据长历史预测，二者条件信息不一致。
监督者也要知道完整历史：让 Teacher 与 Student 使用匹配的长上下文。
  • Teacher 访问长历史，消除 context mismatch
  • Slow Memory 保存稀疏长期语义
  • Fast Memory 保留近期细节与运动
  • 在计算预算内扩大有效监督时域

Fast · Few-step mode

少数大跳

4–8 次网络调用

一次预测一大段噪声区间的平均变化，快速从噪声跳到干净音视频。

  • 输入：带噪当前块 xt、已生成历史 ĉ、区间终点 s、文本/首帧/语音控制

  • 用途：训练时生成历史；部署时直接生成音视频

Slow · Multi-step mode

许多小步

沿 ODE 路径逐小步去噪

每次只预测当前噪声位置下一小段的去噪方向，因此更精细、更稳定。

  • 输入：由真实当前块 x₀ 加噪得到的 xt、快模式历史 ĉ、时间 t 与控制信号

  • 用途：沿许多小步得到高质量轨迹
监督信号：
由真实配对音视频 x₀ 与采样噪声构造的 Flow Matching 目标速度。
闭环快模式让慢模式适应推理分布；慢模式持续变强后，又把更好的能力蒸馏回快模式。

密集保留全部历史既昂贵，也会把错误和旧运动继续传下去。

12 · Consistency / two different axes

先分清：Consistency 在这里有两种含义

一条发生在去噪时间轴，决定能否少步生成；另一条发生在视频时间轴，决定长 rollout 是否连贯。

Diffusion-time consistency
τ

同一内容，不同噪声时刻应指向同一结果

核心问题：如何把 50 个小去噪步压缩成 1–2 个大步？

学生从不同噪声位置出发，都应恢复到同一条教师轨迹对应的干净结果。这是 consistency distillation、CM、ShortCut 讨论的“一致性”。

关注：目标是否正确 · 一步能否跨越轨迹 · 初始化是否稳定
Video-time consistency
t

同一世界，连续帧应保持身份、运动与因果

核心问题：AR 模型持续使用自己的输出后，为什么会漂移？

人物、物体、场景和运动规律应在长时间内保持一致；Prompt 改变时又要及时响应。这是 exposure bias、记忆和 long-context 讨论的“一致性”。

关注：历史分布 · 长程记忆 · 稳定性与动态性的平衡
二者耦合少步误差会进入下一帧历史；时间漂移又会让后续每一次少步去噪变得更难。
12A · Diffusion-time consistency

少步一致性：四个失败模式与当前解法

Problem 1 · Wrong target

教师轨迹不因果

双向 Teacher 使用未来帧；因果 Student 永远无法复现同一 flow map。
解法：先获得 AR Teacher，或直接使用 causal CD 学习正确的 AR 条件映射。
  • Causal Forcing：causal ODE
  • Causal Forcing++：causal CD
目标先对齐，再讨论少步
Problem 2 · Scaling cost

完整 ODE 轨迹太贵

每个样本都要生成、保存几十步 Teacher 轨迹；配置变化后还要重算。
解法：只对齐相邻噪声时刻，用单次在线 Teacher ODE step 提供局部监督。
  • 无需离线轨迹库
  • 每一步优化间隔更小
Causal Forcing++ · causal CD
Problem 3 · One-step geometry

视频轨迹弯得太急

高噪声区域的 trajectory curvature 很大；一步直线跳跃会丢失结构和运动。
解法：保留 2-step 中间锚点，或放弃只追轨迹，直接匹配输出分布。
  • 2-step consistency：跨过弯点
  • One-Forcing：DMD + GAN
一步时需要额外全局真实感约束
Problem 4 · Objective bias

覆盖与锐度难兼得

CM 稳定且覆盖模式，但容易平均化；DMD 更锐利，却易 mode-seeking、依赖初始化。
解法：前向目标负责稳定覆盖，反向目标负责 on-policy 分布细化。
  • Causal-rCM：TF-CM → SF-DMD
  • Mutual：ShortCut + DMD
不是二选一，而是先稳定、再锐化

12B · Video-time consistency

视频一致性：问题都来自“历史不再可靠”

History distribution

训练历史太干净

训练看 GT，推理看自己带误差的输出，导致 exposure bias。
解法：在训练中真正执行少步 rollout，让模型学会从自己的错误继续生成。
  • Self Forcing
  • Mutual Forcing 的 Few-history
  • LongLive 的 long tuning
对齐 train history 与 test history
Teacher horizon

短教师教不了长依赖

短 Teacher 只知道局部窗口，学生却必须根据几十秒历史决策。
解法：学生先走远、教师局部纠错；或者让教师也访问长上下文。
  • Self-Forcing++：长 rollout 抽窗
  • Context Forcing：long-context Teacher
修正 student–teacher context gap
Anchor trade-off

锚点稳定但会冻结运动

固定首帧 Sink 抑制漂移，却可能复制开头、降低动态。
解法：让锚点随历史更新，并把长期、近期、运动记忆分工。
  • Reward：EMA-Sink + Re-DMD
  • Relax：Sink / Tail / Dynamic
  • Deep：Deep Sink + 历史压缩
稳定性与运动活力仍需共同优化
Irreversible history

滑窗走错后无法回头

最近窗口一旦身份漂移，旧场景已被移出，后续只能沿错误轨迹继续。
解法：把远期历史变成可选择、可召回、可遗忘的外部记忆。
  • Echo：场景召回与差异遗忘
  • LongLive-RAG：按内容检索 latent
  • Pyramid：按 head 分配长期历史
从“最近窗口”走向“可寻址记忆”
仍未解决真正的事件级长期记忆、遮挡后的身份恢复、3D 世界状态，以及能区分“稳定”和“静止”的评测指标。
