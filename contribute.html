Contribution guide

Add a discussion or project.

All public content is reviewed through pull requests before it is published.

Discussion directory

Use discussions/YYYY-MM-DD-<leader-short-name>-<topic-slug>/. For example: 2026-07-18-peiyuan-streaming-control.

Pre-meeting contribution

  1. Create a topic branch.
  2. Generate a discussion directory from the template.
  3. Complete metadata, summary, idea map, and references.
  4. Run local validation and preview.
  5. Open a pull request and request review.

Post-meeting contribution

Update the meeting note and action items within 48 hours. Capture insights, changed views, failure cases, connections, and testable next steps.

Media policy

Commit only small, compressed previews. Store large videos externally and link to stable storage. Never publish unpublished research materials without explicit approval.
